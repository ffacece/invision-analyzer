# inVision U: Smart Candidate Analyzer 🚀

Интеллектуальная система глубокого аудита мотивационных эссе для приемной комиссии университета нового поколения. Решение разработано в рамках **Decentrathon 5.0** (трек AI inDrive) для автоматизации первичного отбора кандидатов в inVision U.

Система является **инструментом поддержки принятия решений** (human-in-the-loop). Финальное решение о зачислении всегда остаётся за приёмной комиссией. ИИ не принимает решений об отказе или зачислении автономно.

---

## Ключевые особенности

**Hybrid AI Engine.** Система построена на мультимодельной архитектуре с автоматическим переключением. В качестве основной модели используется `llama-3.3-70b-versatile` для максимально точного анализа. При исчерпании лимитов или ошибке система автоматически переключается на `llama3-8b-8192`. Если обе модели недоступны, возвращается структурированная заглушка — pipeline никогда не падает с ошибкой. Ответ LLM проходит через безопасный парсер, который корректно обрабатывает Markdown-обёртки и невалидный JSON.

**8-критериальная детекция ИИ.** Вместо бинарного ответа «человек или ИИ» система анализирует текст по восьми лингвистическим измерениям: лексическое разнообразие (маркеры ИИ, сленг, живая речь), структура и форматирование (идеальная разбивка, длина абзацев), стилистическая однородность (стабильность тона vs эмоциональные всплески), перплексия и предсказуемость (клише, банальность фраз), фактические особенности (хеджирование, личный опыт, детали), синтаксические паттерны (разнообразие конструкций, ошибки), прагматика (голос автора, ирония, субъективность) и паттерны связности (избыточные коннекторы vs живые переходы). Каждый критерий оценивается от 1 (типично для человека) до 10 (типично для ИИ) с итоговым средним баллом и процентом уверенности.

**Оценка лидерского потенциала.** Система оценивает три ключевых метрики по шкале от 0.0 до 1.0. Leadership — инициатива, влияние на окружающих, организация действий. Motivation — глубина, искренность и внутренний драйв. Growth path — траектория роста, преодолённые трудности, конкретные шаги. Оценка строится на сравнении с «Золотыми стандартами» — эталонными примерами лидерства, встроенными в промпт.

**Трёхуровневая защита Fairness & Privacy.** На первом уровне (Input) система автоматически удаляет из текста персональные данные и protected attributes перед отправкой в LLM. Имена и фамилии заменяются на `[КАНДИДАТ]`, географические названия на `[ЛОКАЦИЯ]`, а также удаляются упоминания возраста, пола, религии, этнической принадлежности, инвалидности, семейного положения и социально-экономического статуса. Для анонимизации имён и локаций используется NER-модель natasha (если установлена) или regex-фолбэк с контекстными маркерами и морфологическими суффиксами. На втором уровне (Prompt) в системном и пользовательском промпте явно прописан запрет учитывать любые демографические характеристики при оценке. LLM оценивает только действия, инициативу и рост. На третьем уровне (Output) ответ LLM сканируется на наличие bias-ключевых слов в объяснении и цитатах. Если обнаружены упоминания protected attributes, результат получает предупреждение `fairness_warning` с пометкой о необходимости ручной проверки.

**Explainability.** Система не является «чёрным ящиком». Для каждой оценки ИИ предоставляет Feature Impact — какой фактор стал решающим и его вес в процентах. Evidence Base — прямые цитаты из эссе, подтверждающие выводы модели. AI Red Flags — конкретные фрагменты текста, вызвавшие подозрение в генерации. Caveats — оговорки, снижающие точность оценки, например при коротком тексте или гибридном авторстве.

## Технический стек

Проект написан на Python 3.10+ с использованием Groq API для доступа к моделям Llama 3.3 70B и Llama 3 8B. Для NER-анонимизации опционально используется библиотека natasha. Данные хранятся и передаются в формате JSON. Конфигурация API-ключа выполняется через python-dotenv и файл `.env`.

Клонируйте репозиторий и установите зависимости:

```bash
git clone https://github.com/your-team/invision-analyzer.git
cd invision-analyzer
pip install groq python-dotenv
pip install natasha  # опционально, для точной NER-анонимизации
