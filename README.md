# inVision U: Smart Candidate Analyzer 🚀

Интеллектуальная система глубокого аудита мотивационных эссе для приемной комиссии университета нового поколения. Решение разработано в рамках **Decentrathon 5.0** (трек AI inDrive) для автоматизации первичного отбора кандидатов в inVision U.

Система является **инструментом поддержки принятия решений** (human-in-the-loop). Финальное решение о зачислении всегда остаётся за приёмной комиссией. ИИ не принимает решений об отказе или зачислении автономно.

---

## Ключевые особенности

**Hybrid AI Engine.** Система построена на мультимодельной архитектуре с автоматическим переключением. В качестве основной модели используется `llama-3.3-70b-versatile` для максимально точного анализа. При исчерпании лимитов или ошибке система автоматически переключается на `llama3-8b-8192`. Если обе модели недоступны, возвращается структурированная заглушка — pipeline никогда не падает с ошибкой. Ответ LLM проходит через безопасный парсер, который корректно обрабатывает Markdown-обёртки и невалидный JSON. При повторяющихся rate-limit ошибках применяется экспоненциальный backoff (1 с → 2 с), после чего происходит переход к следующей модели.

**8-критериальная детекция ИИ.** Вместо бинарного ответа «человек или ИИ» система анализирует текст по восьми лингвистическим измерениям: лексическое разнообразие (маркеры ИИ, сленг, живая речь), структура и форматирование (идеальная разбивка, длина абзацев), стилистическая однородность (стабильность тона vs эмоциональные всплески), перплексия и предсказуемость (клише, банальность фраз), фактические особенности (хеджирование, личный опыт, детали), синтаксические паттерны (разнообразие конструкций, ошибки), прагматика (голос автора, ирония, субъективность) и паттерны связности (избыточные коннекторы vs живые переходы). Каждый критерий оценивается от 1 (типично для человека) до 10 (типично для ИИ) с итоговым средним баллом и процентом уверенности.

**Офлайн-анализ текста.** До отправки в LLM система локально вычисляет семь лингвистических признаков: количество слов и предложений, среднюю длину предложения, Type-Token Ratio (лексическое разнообразие), количество ИИ-маркеров, глаголов конкретных действий и числовых ссылок. Эти признаки передаются в промпт как объективный контекст и сохраняются в итоговом JSON для аудита.

**Оценка лидерского потенциала.** Система оценивает три ключевых метрики по шкале от 0.0 до 1.0. Leadership — инициатива, влияние на окружающих, организация действий. Motivation — глубина, искренность и внутренний драйв. Growth path — траектория роста, преодолённые трудности, конкретные шаги. Оценка строится на сравнении с «Золотыми стандартами» — эталонными примерами лидерства, встроенными в промпт. Принцип оценки: «Show, Don't Tell» — наличие конкретных фактов важнее красочных деклараций.

**Трёхуровневая защита Fairness & Privacy.** На первом уровне (Input) система автоматически удаляет из текста персональные данные и protected attributes перед отправкой в LLM. Имена и фамилии заменяются на `[КАНДИДАТ]`, географические названия на `[ЛОКАЦИЯ]`, организации на `[ОРГАНИЗАЦИЯ]`, а также удаляются упоминания возраста, пола, религии, этнической принадлежности, инвалидности, семейного положения и социально-экономического статуса. Для анонимизации имён и локаций используется NER-модель natasha (если установлена) или regex-фолбэк с контекстными маркерами и морфологическими суффиксами. На втором уровне (Prompt) в системном и пользовательском промпте явно прописан запрет учитывать любые демографические характеристики при оценке. LLM оценивает только действия, инициативу и рост. На третьем уровне (Output) ответ LLM сканируется на наличие bias-ключевых слов в объяснении и цитатах. Если обнаружены упоминания protected attributes, результат получает предупреждение `fairness_warning` с пометкой о необходимости ручной проверки.

**Explainability.** Система не является «чёрным ящиком». Для каждой оценки ИИ предоставляет Feature Impact — какой фактор стал решающим и его вес в процентах. Evidence Base — прямые цитаты из эссе, подтверждающие выводы модели. AI Red Flags — конкретные фрагменты текста, вызвавшие подозрение в генерации. Caveats — оговорки, снижающие точность оценки, например при коротком тексте или гибридном авторстве.

**Кэширование.** Анализатор хранит in-memory кэш (MD5-хэш анонимизированного текста → результат). Повторная отправка одинакового эссе не тратит API-вызов.

## Технический стек

| Компонент | Описание |
|-----------|---------|
| Python 3.14.3 | Язык разработки |
| [OpenRouter API](https://openrouter.ai) | Инференс LLM (meta-llama/llama-3.3-70b-instruct / llama-3.1-8b-instruct) |
| natasha | NER-анонимизация имён, локаций, организаций (опционально) |
| pymorphy2 | Морфологический анализ для regex-фолбэка |
| python-dotenv | Загрузка API-ключа из `.env` |
| JSON | Формат хранения входных и выходных данных |

Полный список зависимостей — в `requirements.txt`.

## Быстрый запуск

### 1. Клонирование и зависимости

```bash
git clone https://github.com/your-team/invision-analyzer.git
cd invision-analyzer
pip install -r requirements.txt
```

Для NER-анонимизации через natasha (рекомендуется):

```bash
pip install natasha
```

Если natasha не установлена, система автоматически переключится на regex-фолбэк — pipeline не упадёт.

### 2. Настройка API-ключа

Создайте файл `.env` в корне проекта:

```env
OPENROUTER_API_KEY=ваш_ключ_от_openrouter
```

Получить ключ можно на [openrouter.ai/keys](https://openrouter.ai/keys).

### 3. Подготовка данных

Отредактируйте `mock_data.json` — добавьте эссе кандидатов. Формат:

```json
[
  {
    "id": "CAND_001",
    "name": "Имя Фамилия",
    "essay": "Текст мотивационного эссе..."
  }
]
```

Поле `name` используется только как идентификатор в выводе и не передаётся в LLM.

### 4. Запуск

```bash
python process_candidates.py
```

Результаты будут выведены в консоль (сводная таблица + карточки explainability) и сохранены в `final_results.json`.

## Алгоритм шортлиста

По завершении анализа система ранжирует кандидатов по **композитному баллу**:

```
composite = (leadership × 0.4 + motivation × 0.3 + growth_path × 0.3) × (1 − ai_risk)
```

Множитель `(1 − ai_risk)` автоматически понижает позицию кандидатов с высокой вероятностью ИИ-генерации эссе. Топ-3 выводятся как рекомендованный шортлист.

## Важные замечания

- Между запросами к API автоматически выдерживается пауза **1.5 секунды** во избежание rate-limit блокировок.
- Файл `.env` не должен попадать в систему контроля версий — добавьте его в `.gitignore`.
- Система не хранит оригинальные тексты эссе в результатах — только анонимизированные цитаты.
- При коротких эссе (<100 слов) уверенность детекции ИИ снижается автоматически, что отражается в поле `caveats`.
